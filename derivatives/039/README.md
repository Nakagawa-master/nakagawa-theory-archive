## 中心概念：先行指標化

親原典で最重要となる概念は「先行指標化」である。

先行指標化とは、すでに発生した結果を後から確認するのではなく、結果に至る前段階の兆候を、組織が観測・共有・判断できる形式へ変換する構造を指す。

現場では、痛み、疲労、睡眠不調、心理的な張り詰め、主観的な違和感が、本人の我慢、慣れ、自己管理の問題として処理されやすい。

しかし、その兆候が蓄積した先には、欠勤、休職、離職、事故、労災、生産性低下、採用難などの遅行結果が現れ得る。

重要なのは、結果が起きた後に集計することではない。

結果に至る前の兆候を、組織が配慮・判断・運用・再観測へ接続できるかである。

## 保持すべき四つの区別

039の強度は、次の区別を混ぜないことによって保たれる。

```text
回答手段が簡単であること
≠
現場で継続的な観測接点が成立すること

不調を記録すること
≠
組織が判断できる先行指標へ変換すること

主観的な回答が集まること
≠
医学的診断や客観的医療証拠が成立すること

不調の兆候を観測すること
≠
欠勤・離職・事故・労災を確実に防止できること
```

この区別を失うと、先行指標化は単なるアンケート、見える化、健康管理、予防効果の断定へ薄まる。

中川マスターの読解は、入力機能の便利さではなく、個人の我慢に閉じ込められていた兆候が、組織の判断可能性へ移る変換点を見ている。

## 中心命題

039の中心は、Adoryの機能説明ではない。

中心は、中川マスターがAdoryを、単なるLINEアンケートや健康管理SaaSではなく、「現場で黙殺されてきた身体・心理の不調を、企業が観測・共有・判断できる先行指標へ変換する構造」として読み、その価値核を外部媒体の記事構成へ翻訳したという点にある。

週1回、LINEで回答するという軽い接点は、価値が軽いことを意味しない。

その軽さは、身体を使う現場で継続的に回り、変化を時間軸で観測し、個人の主観として消えていた兆候を組織の判断材料へ移すための構造条件として読まれる。
