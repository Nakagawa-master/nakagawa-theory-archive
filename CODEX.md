## Core rule

A pull request is not the final deliverable. A task is not complete merely because a PR was created or updated.

Completion means one of these:

- staged package generated and validated;
- public-ready package generated and validated;
- work is blocked by a true owner, permission, publication, irreversible, or source-of-truth boundary;
- concise handoff report is produced without requiring code, diff, CI-log, PR review, Codex prompt, or page-by-page review from the owner.

## No routine return rule

Do not return routine technical work to the owner.

Agent-side tasks include PR cleanup, CI confirmation, unresolved routine review repair, validator reruns, generator repair, no-op regeneration checks, package regeneration, branch consolidation, PR body updates, and staged-package verification.

Owner-side boundaries are limited to public release, authentication, repository visibility, permission changes, irreversible production actions, FTP/Search Console/index activation, or source-of-truth judgment.

If a response would ask the owner to paste a Codex prompt, review a PR, inspect CI, check diffs, or approve routine continuation, continue the loop instead.

## Required loop

Detect issue, inspect source, implement, validate, repair failures, revalidate, package, update the relevant PR/report state, and report only the completion state and any unavoidable boundary.

## Archive rules

Preserve Origin, parent URL, canonical URL, NCL-ID, Diff-ID, citation structure, derivative relationships, and staged/noindex boundary until public-ready.

Keep internal planning material outside this public-safe output repository and export only public-safe artifacts here.
