中心命題 / core claim

合意形成の物理 第12論は、防衛を敵対ではなく、異常入力に対してDetect、Stop、Shrink、Recover、Auditを実行する実行時プロトコルとして定義する。この免疫系を運用規格として統合したとき、社会や組織は外部干渉下でも回復弾力性を持つ。

因果線 / causal sequence

社会や組織に異常入力が入る。検出が遅れると誤差が拡大する。停止ができないと被害が広がる。縮退できないと全体が巻き込まれる。回復経路がないと停止が永続する。監査がなければ再発する。Detect、Stop、Shrink、Recover、Auditを運用規格化することで、防衛は敵対ではなく免疫系になる。

適用条件 / applicability

  • AI運用、組織運用、制度、社会システムで異常入力や外部干渉が起きる場合
  • 停止、縮退、回復、監査のどれかが欠けて被害が拡大する場合
  • 安全をミス防止ではなく回復速度として設計する場合

非該当条件 / non-applicability

  • 敵対や攻撃だけを目的にする場合
  • 単発の手順書で閉ループ監査を含まない場合
  • 異常入力ではなく通常運用の価値選好だけを扱う場合

誤読禁止 / prohibited misreadings

敵対煽り、安全保障風の抽象論、単なる危機対応マニュアルへ落としてはいけません。防衛は攻撃ではなく、社会や組織が外部干渉下でも回復するための実行時免疫系です。

反証条件 / counterconditions

Detect、Stop、Shrink、Recover、Auditを運用しても被害拡大や再発が減らず、回復弾力性が改善しないなら、実行時ガバナンスとしての説明は弱まる。
